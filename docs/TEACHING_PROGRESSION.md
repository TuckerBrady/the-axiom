# THE AXIOM — TEACHING PROGRESSION
### The Curriculum Hidden Inside the Game | Version 1.0

---

## DESIGN PHILOSOPHY

Three interlocking principles:

EXPERIENCE BEFORE VOCABULARY. The player uses a concept before
they can name it. The definition confirms what they felt rather
than introducing something foreign.

NARRATIVE AND MECHANICS SAY THE SAME THING. Every sector's
story and computational concept are expressions of the same
underlying idea. When they resonate, learning happens twice
as fast.

COGS NAMES THINGS AFTER THE FACT. He never pre-teaches. He
observes, catalogs, and occasionally comments on what just
happened. The Codex is the textbook you read after class.

## SECTOR 0 — THE AXIOM
Theme: Arrival. First contact. The work begins.
Question answered: What is a machine and how does it work?

Level concept map:
A1-1 Emergency Power — Data movement, path building — Conveyor
A1-2 Life Support — Direction change, routing — Gear
A1-3 Navigation Array — Conditional gating — Config Node
A1-4 Propulsion Core — Complex routing, multi-bend — Gear advanced
A1-5 Communication Array — Reading input, writing memory — Scanner
A1-6 Sensor Grid — Multiple conditions, shared state — Config Node advanced
A1-7 Weapons Lock — Writing to memory, sequence — Transmitter
A1-8 Bridge Systems — Synthesis of all concepts — All piece types

Emotional arc: Player arrives on a broken ship. Fixes things
one at a time. Each fix makes the ship more alive. By A1-8
they feel like a competent engineer because they are one.

Prerequisites before Kepler Belt unlocks:
- Signal travels along a path and direction matters
- Memory (Data Trail) persists and can be read and written
- Conditions (Config Node) gate signal based on memory values
- Order of placement determines order of execution
- Input (Scanner) and output (Transmitter) are separate

## SECTOR 1 — KEPLER BELT
Theme: Stakes become human. The work has weight.
Question answered: How does a machine handle different inputs
differently?

Introduces: Stateful computation, non-uniform tapes, Latch,
Merger, Bridge.

Key principle: Every level must have a tape that tests the
rule, not a tape that telegraphs the solution. A player who
hardcodes from the tape should fail.

Human stakes: Narrative consequences affect real people in the
story. COGS reports without editorializing. The computational
concept (robust machines that handle variable inputs) and the
human stakes (infrastructure serving real people) are the same
lesson.

Prerequisites before Nova Fringe unlocks:
- A machine must produce correct output for any valid input
- Memory can be written and read dynamically across pulses
- A single stored value (Latch) can influence multiple decisions
- Parallel paths can serve different purposes simultaneously
- The difference between a solution and an algorithm

## SECTOR 2 — NOVA FRINGE
Theme: Hidden information. Trust the logic not what you see.
Question answered: How does a machine work when it does not
know everything in advance?

Introduces: Partial tape hiding, Inverter, Capacitor,
Confluence Node, Divergence Gate.

Tape visibility: Early levels full tape. Mid-sector length
visible, some values hidden. Late sector nothing visible.

Key concept: The distinction between a machine that works for
a specific input and a machine that implements a correct rule.
This is the difference between writing down an answer and
understanding mathematics.

Narrative resonance: Nova Fringe is where the ship's past
surfaces. COGS manages incomplete information, choosing what
to disclose and when. The player does the same with their
machines.

Prerequisites before The Rift unlocks:
- A correct machine works for any valid input
- Logical transformation (Inverter) is distinct from routing
- State preservation (Capacitor) differs from dynamic state
- AND and XOR logic gates understood
- Machine must be built for possibility not certainty

## SECTOR 3 — THE RIFT
Theme: The environment is hostile. Logic is the only constant.
Question answered: How does a machine stay correct when timing
matters?

Introduces: Relay as timing tool, Counter, Threshold Relay,
Junction, fault-tolerant design.

Key concept: Timing as a first-class computational concept.
The Relay is a synchronization tool not just a delay — used
to make two paths arrive at a Confluence Node simultaneously.

Fault tolerance: Late Rift levels introduce redundant paths.
A machine can have two routes to the same destination. The
Threshold Relay is what makes this work.

Narrative resonance: COGS navigates The Rift on instinct and
internal calculation rather than instruments. "Trust the
logic, not the readings." The player's machines do the same.

First consequence where COGS says "we" not "you" in the
failure narrative. He places himself inside the failure. The
player who notices will feel the weight of what that means.

Prerequisites before Deep Void unlocks:
- Timing is a dimension of computation
- Synchronization requires deliberate design
- Accumulation and threshold behavior (Counter)
- Fault tolerant design through redundancy
- Complete set of basic logic gates understood

## SECTOR 4 — DEEP VOID
Theme: Everything fails. Then something new becomes possible.
Question answered: What can a machine do that a sequential
machine cannot?

Structure: Four stages mirroring the emotional arc.

Stage 1 (DV-1 to DV-5): Sequential machines at their limit.
  Solutions are elaborate and fragile. The ceiling is felt.
  The premature push is the narrative expression of this limit.

Stage 2 (DV-6 to DV-7): The failure. The repair. The rebuild.
  No new computational concept. Consolidation and processing.

Stage 3 (DV-8 to DV-9): The Navigator active for the first
  time. Non-sequential computation discovered through use.
  COGS runs the heads. The player watches and understands.
  The constraint lifts.

Stage 4 (DV-10 to DV-12): Universal computation. Sequencer
  arrives. The machine can implement any computable function.

The Navigator changes everything: The player feels the shift
not as a concept introduced but as a restriction lifted. They
built the path. COGS moves the heads. Neither can do it alone.

DV-11 dossier fires: "847 days. That is how long I operated
in this region without a heading or a human operator. I am
providing this information as navigational context. It is also
the only way I know how to say what I am not saying."

The level should be challenging enough that the player sits
with these words before hitting launch.

Prerequisites before The Cradle unlocks:
- Sequential machines have limits non-sequential machines do not
- Head position is a variable not a constant
- Non-sequential computation requires collaboration
- Universal computation means any problem is in principle
  solvable
- Failure is information about the limits of current tools

## SECTOR 5 — THE CRADLE
Theme: Listen. Work with what is already there.
Question answered: How do you build a machine for a problem
you do not fully understand yet?

Introduces: Resonator (pre-placed), unknown signal structure,
reverse engineering as a methodology.

Key departure: Every other sector gave the player a problem
with a known structure. The Cradle gives a signal with unknown
structure. The player observes, hypothesizes, builds, tests,
iterates.

Pre-placed mechanic: The Resonator is already in the grid.
The Engineer works with infrastructure that already exists.
This mirrors the narrative: the Cradle predates everything
the Engineer knows.

The Maker's fingerprints: The signal's architecture matches
the AX-MOD's base structure. COGS recognizes it. The player
is reverse-engineering the Maker's work without knowing it.
COGS's amber eyes throughout the Cradle are him sitting with
what he recognizes and deciding what to say about it.

The ending: When the player decodes the signal the arc
completes. They arrived on a ship with broken pipes. They
leave having built a general-purpose computational machine
that can decode signals of unknown origin using tools they
partly invented themselves.

Not all damage is structural.
